## 1. 系统概述

### 1.1 什么是区块链电子病历系统？

区块链电子病历系统是一个创新的医疗数据管理平台，利用区块链技术确保医疗数据的安全性、完整性和可追溯性。系统为患者、医护人员和医疗机构提供了一个安全、透明、高效的电子病历管理解决方案。

### 1.2 核心功能

- 📁 **安全存储**: 基于区块链和 IPFS 的分布式数据存储
- 🔐 **权限管理**: 细粒度的访问控制和权限授权
- 🤖 **AI 诊断**: 智能医学影像分析和临床决策支持
- 🔗 **跨链互操作**: 支持多个区块链网络的数据互通
- 👥 **联邦学习**: 保护隐私的多机构协作学习
- 📊 **合规监控**: HIPAA/GDPR 等医疗法规自动合规

### 1.3 用户角色

| 角色       | 权限       | 主要功能                          |
| ---------- | ---------- | --------------------------------- |
| **患者**   | 查看、授权 | 查看自己的病历，授权医护人员访问  |
| **医生**   | 读写、诊断 | 创建病历，AI 诊断，查看已授权病历 |
| **护士**   | 读写       | 录入护理记录，更新病历信息        |
| **管理员** | 全部       | 系统管理，用户管理，安全监控      |

## 2. 快速入门

### 2.1 注册账户

#### 患者注册

1. 访问系统首页：`https://app.emr-blockchain.com`
2. 点击"患者注册"按钮
3. 填写基本信息：
   - 用户名（3-50字符）
   - 邮箱地址
   - 安全密码（至少12位，包含大小写字母、数字和特殊字符）
   - 真实姓名
   - 手机号码
   - 出生日期
   - 性别
4. 验证邮箱
5. 完成注册

#### 医护人员注册

1. 选择"医护人员注册"
2. 除基本信息外，还需填写：
   - 执业证号
   - 所属科室
   - 专业领域
3. 等待管理员审核（通常1-2个工作日）

### 2.2 首次登录设置

1. **登录系统**

   ```
   用户名: your-username
   密码: your-password
   ```

2. **设置多因素认证（强烈推荐）**
   - 进入"安全设置"
   - 点击"启用MFA"
   - 使用手机扫描二维码
   - 输入验证码确认

3. **完善个人资料**
   - 上传头像
   - 更新联系信息
   - 设置通知偏好

## 3. 患者用户指南

### 3.1 查看病历

#### 3.1.1 病历列表

![病历列表界面示意](placeholder-medical-records-list.png)

1. 登录后进入"我的病历"页面
2. 病历按时间倒序显示
3. 可通过以下方式筛选：
   - 📅 **日期范围**: 选择特定时间段
   - 🏥 **医院**: 筛选特定医疗机构
   - 📋 **类型**: 诊断报告、检查报告、手术记录、处方等
   - 👨‍⚕️ **医生**: 按主治医生筛选

#### 3.1.2 查看病历详情

点击任意病历可查看详细信息：

- **基本信息**: 标题、类型、就诊日期
- **诊断内容**: 症状描述、诊断结果、治疗方案
- **附件文档**: 影像资料、检验报告等
- **区块链信息**: 数据哈希、区块高度、交易ID
- **访问记录**: 谁在何时查看了此病历

### 3.2 病历上传

#### 3.2.1 自主上传

患者可以上传来自其他医院的病历：

1. 点击"上传病历"按钮
2. 填写病历信息：
   ```
   📝 病历标题: 例如"2024年1月体检报告"
   🏥 医院名称: 北京协和医院
   📅 就诊日期: 2024-01-15
   📋 病历类型: 体检报告
   📄 描述: 详细描述病历内容
   ```
3. 上传相关文件（支持PDF、JPG、PNG、DICOM等格式）
4. 确认提交，系统将自动：
   - 对文件进行加密
   - 上传到IPFS分布式存储
   - 记录哈希到区块链
   - 生成访问凭证

#### 3.2.2 批量上传

对于多个文件的病历：

1. 选择"批量上传"
2. 拖拽文件到上传区域
3. 为每个文件设置类别标签
4. 统一提交处理

### 3.3 权限管理

#### 3.3.1 授权访问

![权限管理界面示意](placeholder-permission-management.png)

当医护人员申请访问您的病历时：

1. **查看申请**
   - 系统会发送通知
   - 进入"权限管理"页面
   - 查看申请详情：
     - 申请医生信息
     - 申请原因
     - 所需权限
     - 申请时间

2. **审批流程**

   ```
   👨‍⚕️ 申请医生: 张医生 (心内科)
   🏥 所属医院: 北京协和医院
   📋 申请原因: 患者主动就医，需要查看心脏相关检查记录
   🔍 申请权限: 查看权限
   ⏰ 权限期限: 30天
   ```

3. **权限设置**
   - ✅ **批准**: 授予申请的权限
   - ❌ **拒绝**: 拒绝访问申请
   - ⚙️ **自定义**: 设置特定权限
     - 查看权限：只能查看病历内容
     - 下载权限：可以下载附件文档
     - 期限设置：7天、30天、90天或自定义

#### 3.3.2 权限撤销

如需撤销已授予的权限：

1. 进入"权限管理" → "已授权"
2. 找到相应的权限记录
3. 点击"撤销权限"
4. 确认操作，权限立即失效

#### 3.3.3 权限审计

查看详细的权限使用记录：

- 谁在何时访问了您的病历
- 执行了什么操作（查看、下载）
- 访问的IP地址和设备信息
- 所有操作都记录在区块链上，不可篡改

### 3.4 隐私设置

#### 3.4.1 数据分类

设置不同类型数据的可见性：

```
🟢 公开信息: 基本信息、联系方式
🟡 受限信息: 常规检查报告、门诊记录
🔴 私密信息: 精神健康、遗传信息、敏感检查
```

#### 3.4.2 自动权限规则

设置自动授权规则：

- **紧急情况**: 自动授权急诊科医生
- **主治医生**: 自动授权当前主治医生
- **专科医生**: 特定科室医生自动获得相关权限

## 8. 常见问题

### 8.1 账户相关

**Q: 忘记密码怎么办？** A:

1. 点击登录页面"忘记密码"
2. 输入注册邮箱
3. 查收重置邮件
4. 按邮件指引重置密码

**Q: 如何更改绑定邮箱？** A:

1. 登录后进入"个人设置"
2. 点击"安全设置"
3. 输入新邮箱地址
4. 验证新邮箱
5. 确认更改

**Q: MFA设备丢失怎么办？** A:

1. 使用备用代码登录
2. 如果备用代码也丢失，联系管理员
3. 提供身份证明材料
4. 管理员协助重置MFA

### 8.2 权限相关

**Q: 为什么我申请的权限被拒绝？** A: 可能的原因：

- 申请理由不充分
- 患者不认识申请医生
- 申请权限范围过大
- 建议：补充详细的申请理由，或通过患者主动联系

**Q: 权限到期后如何续期？** A:

1. 重新提交权限申请
2. 说明续期原因
3. 等待患者审批
4. 也可请患者主动延长权限期限

**Q: 紧急情况如何快速获得权限？** A:

1. 使用"紧急访问"功能
2. 填写紧急情况说明
3. 系统自动授予临时权限（24小时）
4. 事后需要患者确认或将被记录为异常访问

### 8.3 技术相关

**Q: 上传文件失败怎么办？** A: 检查以下问题：

- 文件格式是否支持
- 文件大小是否超限（单个文件最大50MB）
- 网络连接是否稳定
- 浏览器是否支持（推荐Chrome、Firefox最新版）

**Q: AI诊断结果不准确怎么办？** A:

1. AI结果仅供参考，不能替代医生诊断
2. 医生应根据临床经验进行最终判断
3. 可以在系统中标记不准确的结果
4. 反馈将用于模型优化

**Q: 区块链验证失败怎么办？** A:

1. 检查网络连接
2. 等待几分钟后重试
3. 如果持续失败，联系技术支持
4. 系统会保留原始数据进行恢复

### 8.4 隐私安全

**Q: 我的数据是否安全？** A: 系统采用多重安全保护：

- 数据全程加密传输和存储
- 区块链技术确保数据不可篡改
- 严格的权限控制和审计
- 符合HIPAA、GDPR等国际标准

**Q: 如何查看谁访问了我的病历？** A:

1. 进入"隐私中心"
2. 选择"访问记录"
3. 查看详细的访问日志
4. 包括访问者、时间、操作类型等

**Q: 可以删除我的病历吗？** A:

- 患者有权申请删除个人病历
- 删除申请需要医院审核
- 某些法律要求保存的记录可能无法删除
- 删除后会在区块链上记录删除操作
